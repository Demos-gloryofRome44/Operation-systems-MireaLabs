# Лабораторная работа №7: Межпроцессное взаимодействие через разделяемую память

## Цель работы
Ознакомление с механизмами межпроцессного взаимодействия (IPC) в операционных системах, изучение работы с разделяемой памятью и семафорами для синхронизации процессов.

## Описание задачи
Разработать две независимые программы, которые взаимодействуют через разделяемую память:
1. **Программа-отправитель** (sender) - передает текущее время и свой PID
2. **Программа-получатель** (receiver) - принимает данные и выводит их вместе со своим временем и PID

## Технические детали реализации

### Используемые технологии
- **Разделяемая память** (Shared Memory) - для обмена данными между процессами
- **Семафоры POSIX** - для синхронизации доступа к разделяемой памяти
- **Сигналы** - для корректного завершения программ (Ctrl+C)
- **Динамическое выделение памяти** через `mmap()`

### Структура данных в разделяемой памяти
```c
typedef struct {
    char time_str[64];  // Строка с временем
    int pid;            // PID процесса-отправителя
} shared_data_t;
```

### Механизм синхронизации
Используется два семафора:
1. **sem_sender** - управляет доступом отправителя к разделяемой памяти
2. **sem_receiver** - управляет доступом получателя к разделяемой памяти

Последовательность работы:
1. Отправитель ждет семафор `sem_sender`
2. Отправитель записывает данные в разделяемую память
3. Отправитель освобождает семафор `sem_receiver`
4. Получатель ждет семафор `sem_receiver`
5. Получатель читает данные из разделяемой памяти
6. Получатель освобождает семафор `sem_sender`


## Инструкция по сборке и запуску

### Сборка программ
```bash
# Очистка предыдущих сборок
make clean

# Сборка всех программ
make
```

### Запуск программ

1. **Запуск отправителя** (в отдельном терминале):
```bash
./sender
```

2. **Запуск получателей** (в одном или нескольких терминалах):
```bash
./receiver
```
